# Fix Expo SDK 54 Version Mismatch

## Problem
App crashes with: `TypeError: 0, _reactNative.useEffect is not a function (it is undefined)`

**Root cause**: React/React Native version mismatch. React Native 0.81.5 requires React 19, but the current package.json has React 18.3.1.

## Current State (Broken)
- `react-native`: 0.81.5  ← Needs React 19
- `react`: 18.3.1      ← Wrong version
- `expo`: ~54.0.33     ← SDK 54, expects React 18-19 compatible RN

## Solution
**Restore React 19.x** with React Native 0.81.5 (the original working combination).

Expo CLI warnings about package versions are expected and safe to ignore - they're just suggestions from the compatibility checker. The actual runtime error is caused by the React version mismatch.

## Steps to Fix

### 1. Restore package.json to correct versions
Edit package.json dependencies:
```json
"react": "19.1.0",
"react-dom": "19.1.0",
"react-native": "0.81.5",
```
Remove any accidental changes: expo-application, expo-device if present.

### 2. Clean reinstall
```bash
# Delete node_modules and lockfile
rm -rf node_modules package-lock.json

# Clean npm cache
npm cache clean --force

# Install with legacy peer deps to bypass version conflicts
npm install --legacy-peer-deps
```

### 3. Clear Metro bundler cache
```bash
npx expo start --clear
```

### 4. Test the app
- Press `a` to open on Android emulator
- OR scan QR with Expo Go app

### 5. Verify build
```bash
npx expo export --platform android
```
Should complete successfully with no errors.

## Why This Works
- Expo SDK 54 is built for React Native 0.81.x
- React Native 0.81.x has peer dependency: `"react": "^19.1.0"`
- React 19.1.0 + RN 0.81.5 + Expo 54 = compatible
- The error `useEffect is not a function` occurs when React 18 is paired with RN 0.81.5 because RN expects React 19's internal hooks implementation

## Common Issues & Solutions

### If still seeing the same error:
1. Verify installed versions:
   ```bash
   npm ls react react-native expo
   ```
   Should show:
   - react@19.1.0
   - react-native@0.81.5
   - expo@~54.0.33

2. If wrong versions persist, force reinstall:
   ```bash
   npm install react@19.1.0 react-dom@19.1.0 react-native@0.81.5 --legacy-peer-deps
   ```

3. Clear Expo cache completely:
   ```bash
   rm -rf .expo
   npx expo start --clear
   ```

### If build fails with metro errors:
```bash
# Reset Metro cache location
npx expo start --clear --reset-cache
```
